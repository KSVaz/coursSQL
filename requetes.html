Les requêtes ne sont pas sensibles à la casse, mais !!!

Une ocnvention indique qu'il faut mettre les mots-clefs des requêtes en majuscule.

  • Ces deux requêtes sont identiques (* = all, tout) :
  • SELECT id_employes, prenom, nom, sexe, service, date_embauche, salaire FROM employes;

Les conditions : le mot-clef WHERE

Nous cherchons avec une condition une partie des données

Affiche-moi le nom et le prénom de la table des employés à condition que les employés fassent parti du service informatique

  • SELECT nom, prenom FROM employes WHERE service='informatique';
  • Ici on demande d'afficher les informations des employées dont le sexe est féminin : SELECT * FROM employes WHERE sexe='f';

Le mot-clef BETWEEN

Afficher les employés ayant été recrutés entre 2006 et 2010

  • SELECT * FROM employes WHERE date_embauche BETWEEN '2006-01-01' AND '2010-12-31';
  • SELECT nom, prenom, date_embauche FROM employes WHERE date_embauche BETWEEN '2006-01-01' AND '2010-12-31';
  • Dans une BDD la date est inversée : YYYY/MM/JJ
  • Une requête qui donne la date au serveur, la date du jour : SELECT CURDATE(); (curdate = current date)
  • Entre une date et aujourd'hui : SELECT * FROM employes WHERE date_embauche BETWEEN '2006-01-01' AND CURDATE();

Le mot-clef LIKE

Afficher les employés dont le prénom commence par S, ce qui sera très utile dans un moteur de recherche

Avec la présence des signes de pourcentages avant et après le caractère ou le mot recherché, nous demandons au système de trouver les enregistrements contenant l'expression recherchée.

  • SELECT * FROM employes WHERE prenom LIKE 's%'; (le pourcentage signifie une suite de caractères)

    • Afficher les employés dont le prénom termine par S

  • SELECT * FROM employes WHERE prenom LIKE '%s';

    • Afficher les employés dont le prénom contient la lettre s

  • SELECT * FROM employes WHERE prenom LIKE '%s%';

II. Les opérateurs de comparaison

Opérateur Symbole
= est égal
> strictement supérieur
< strictement inférieur
>= supérieur ou égal
<= inférieur ou égal
< > ou != est différent

L'opérateur !=

  • Tous les employés qui ne sont pas au service informatique : SELECT * FROM employes WHERE service!='informatique';

L'opérateur >

  • Tous les employés gagnant plus de 3000 euros : SELECT nom, prenom, service, salaire FROM employes WHERE salaire > 3000;

III. Le classement ORDER BY

  • Les employés triés par nom de famille et par ordre alphabétique : SELECT prenom, nom FROM employes ORDER BY nom;
  • Classement par ordre ascendant (par défaut donc d plus petit au plus grand) : SELECT prenom, nom FROM employes ORDER BY nom ASC;
  • Classement par ordre descendant : SELECT prenom, nom FROM employes ORDER BY nom DESC;
  • ---
  • Trier sur plusieurs colonnes : SELECT prenom, nom, service, salaire FROM employes ORDER BY service ASC, salaire ASC;

IV. Limiter les résultats avec le mot-clef LIMIT

  • Afficher les employés en les classant par salaire (du + grand au plus petit) et en les affichant par 3SELECT * FROM employes ORDER BY salaire DESC LIMIT 0,3;
  • Le premier chiffre précise l'enregistrement de départ (0 définit le premier enregistrement en informatique), le second chiffre nous indiquele nombre de résultats que l'on souhaite obtenir
  • Autre exeple : SELECT * FROM employes ORDER BY salaire DESC LIMIT 6,3;
  • Le mot-clef ou la clause LIMIT est utile pour une pagination web (exemple : afficher 40 éléments par page)

V. Réaliser des calculs dans nos requêtes

  • Afficher la liste des employés avec leur salaire annuel : SELECT prenom, nom, service, salaire, salaire*12 FROM employes ORDER BY salaire*12;

Utiliser un alias AS

  • SELECT prenom, nom, salaire, salaire*12 AS salaire_annuel FROM employes ORDER BY salaire_annuel DESC;
  • Le mot-clef AS permet de définir un alias
  • Calculer le salaire par heureSELECT nom, prenom, salaire, salaire/150 AS salaire_heure FROM employes ORDER BY salaire_heure ASC;

Calculer une somme avec SUM et une moyenne avec AVG

  • Faire une somme, une addition : SELECT SUM(salaire*12) FROM employes;
  • AVG faire la moyenne (average)SELECT AVG(salaire*12) FROM employes; ou la moyene mensuelle : SELECT AVG(salaire) FROM employes;

Arrondir avec ROUND

  • Arrondir un calcul : SELECT ROUND(AVG(salaire)) FROM employes;
  • Faire un alias : SELECT ROUND(AVG(salaire)) AS moyenne_mensuelle FROM employes;

Compter avec COUNT

  • Compter le nombre d'employés : SELECT COUNT(*) FROM employes WHERE sexe='f';

VI. Isoler une valeur minimum/maximum avec MIN ou MAX

  • Trouver le salaire minimum : SELECT MIN(salaire) FROM employes;
  • Cette requête ne fonctionnera pas : SELECT prenom, MIN(salaire) FROM employes; Pour trouver le salaire le plus bas et la personne qui gagne ce salaire il faut écrire SELECT prenom, salaire FROM employes WHERE salaire=(SELECT MIN(salaire) FROM employes); Ici, le système va d'abord exécuter la requête entre parenthèse pour connaître le résultat et l'utiliser comme source pour l'autre requête.
  • Trouver le salaire maximum : SELECT MAX(salaire) FROM employes;
  • Trouver qui gagne le plus : SELECT prenom, salaire FROM employes WHERE salaire=(SELECT MAX(salaire) FROM employes); On appelle cela une requête imbriquée
  • Qui gagne 1390 EurosSELECT prenom, nom, salaire FROM employes WHERE salaire=1390;

VII. Condition IN avec plusieurs valeurs

  • Trouver les employés à la comptabilité et au service informatique : SELECT prenom, nom, service FROM employes WHERE service IN ('comptabilite', 'informatique');
  • ...
  • = (égal) permet d'annoncer une seule valeur
  • IN permet d'annoncer plusieurs valeurs
  • ...
  • Trouver les employés qui ne sont pas au service informatique et à la comptabilité : SELECT nom, prenom, service FROM employes WHERE service NOT IN ('comptabilite', 'informatique') ORDER BY service;

VIII. Conditions multiples, différents champs et valeurs

  • Trouver les personnes du service commercial dont le salaire est inférieur ou égal à 2000 Euros : SELECT nom, prenom, service, salaire FROM employes WHERE service='commercial' AND salaire <= 2000;

Ordre de priorité sur les conditions AND et OR

  • SELECT nom, prenom, service, salaire FROM employes WHERE service='production' AND salaire=1900 OR salaire=2300;
  • Lorsqu'il y a AND et OR au sein de la même requête, le système lit la condition de la droite vers la gauche et il verra d'abord salaire='1900' OR salaire='2300' et ensuite service='production'
  • En revanche avec cette requête : SELECT nom, prenom, salaire FROM employes WHERE salaire IN (1900, 2300) AND service='production';
  • Pour garantir l'ordre naturel de lecture (de la gauche vers la droite) on utilisera les parenthèses : SELECT prenom, nom, service, salaire FROM employes WHERE service='production' AND (salaire=1900 OR salaire=2300);

XI. Le regroupement avec GROUP BY

  • On veut connaitre le nombre d'employés par service : SELECT service, COUNT(*) AS nombre_employes FROM employes GROUP BY service;

Utilisation du mot-clef HAVING, ayant

  • On impose une condition COUNT et HAVINGSELECT service, COUNT(*) AS nombre_employe FROM employes GROUP BY service HAVING COUNT(*) > 2;

X. Les requêtes d'insertion avec INSERT

  • Insérer un enregistrement : INSERT INTO employes(prenom, nom, sexe, service, date_embauche, salaire) VALUES('Alexis', 'Richy', 'm', 'informatique', '2011-12-28', 1800);
  • Insérer un enregistrement : 2nd méthode INSERT INTO employes VALUES(NULL, 'Jean', 'Richy', 'm', 'informatique', '2012-01-28', 1800); NULL nous permet de laisser le champs id_employes se remplir seul car il est auto-incrémenté.

XI. Les requêtes de modification avec UPDATE

La requête UPDATE

  • Augmenter le salaire d'un employé : UPDATE employes SET salaire = 1871 WHERE id_employes=990;
  • Il est recommandé de modifier un enregistrement en ciblant son id, l'id étant unique.
  • ...
  • Pour modifier plusieurs colonnes respectons l'ordre des colonnes : UPDATE employes SET service = 'commercial', salaire = 1999 WHERE id_employes = 990;

La requête DELETE

  • Supprimer un enregistrement : DELETE FROM employes WHERE id_employes = 388;
  • Supprimer plusieurs informations, plusieurs enregistrements : DELETE FROM employes WHERE service='informatique';

XII. Exercice

  1. Afficher la profession de l'employé 547 : SELECT service FROM employes WHERE id_employes=547;
  2. Afficher la date d'embauche d'Amandine : SELECT prenom, date_embauche FROM employes WHERE prenom='Amandine';
  3. Afficher le nombre de commerciaux : SELECT COUNT(*) AS 'nbr_commerciaux' FROM employes WHERE service ='commercial';
  4. Afficher le coût des commerciaux sur une année : SELECT SUM(salaire*12) AS 'salaire_annuel_commercial' FROM employes WHERE service='commercial'
  5. Afficher le salaire moyen par service : SELECT AVG(salaire) FROM employes GROUP BY service;
  6. Afficher le nombre de recrutement sur l'année 2010 : SELECT date_embauche, COUNT(*) AS 'nbr de recrutement 2010' FROM employes WHERE date_embauche BETWEEN '2010-01-01' AND '2010-12-31' GROUP BY date_embauche; ou SELECT COUNT(*) AS 'nbr de recrutement 2010' FROM employes WHERE date_embauche LIKE '2010%'; ou SELECT COUNT(*) as 'nb de recrutement' FROM employes WHERE date_embauche >= '2010-01-01' AND date_embauche <= '2010-12-31';
  7. Augmenter le salaire pour chaque employé de 100€ : UPDATE employes SET salaire=salaire+100;
  8. Afficher le nombre de service : SELECT COUNT(DISTINCT service) AS 'nombre de services' FROM employes;
  9. Afficher les informations de l'employé du service commercial gagnant le salaire le plus élevé : SELECT * FROM employes WHERE service ='commercial' AND salaire=(SELECT MAX(salaire) FROM employes WHERE service='commercial');
  10. Afficher l'employé ayant été embauché en dernier : SELECT * FROM employes WHERE date_embauche=(SELECT MAX(date_embauche) FROM employes);
